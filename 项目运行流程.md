# 项目运行流程

## 1. 项目目标

这个项目的核心目标是：

- 每天自动生成一篇技术相关内容
- 将内容保存到本地 Git 仓库
- 自动提交并推送到 GitHub
- 保持 GitHub contribution 活跃记录

生成的内容可以是：

- 算法题解析
- Python 技巧
- Web 开发内容
- 系统设计文章
- 新技术介绍
- 其他技术主题

## 2. 首次使用流程

首次运行时，需要先完成基础配置，主要包括两部分：

- AI 配置
- GitHub 配置

需要准备的信息：

- AI 服务商或兼容 OpenAI 的接口地址
- API Key
- 模型名称
- GitHub 仓库地址
- GitHub 认证方式
- 本地仓库存放目录

可通过下面命令进入初始化流程：

```bash
py -3 main.py config
```

也可以直接打开图形界面，在“配置”页手动填写。

## 3. 日常运行流程

程序启动后，整体流程如下：

1. 读取 `config.json`
2. 初始化 AI 客户端
3. 初始化 GitHub 仓库管理器
4. 根据主题设置生成文章内容
5. 将文章写入本地仓库
6. 自动执行 `git add / commit / push`
7. 更新界面状态和日志

## 4. GUI 使用流程

桌面程序主要分为三个页面：

- 签到
- 配置
- 设置

### 签到页

可以执行：

- 立即签到
- 预览生成内容但不推送
- 选择主题
- 设置单次生成文章数量
- 查看运行日志

### 配置页

用于维护：

- AI 服务商
- Base URL
- 模型名称
- API Key
- GitHub 仓库地址
- GitHub 认证方式
- 本地仓库目录

### 设置页

用于维护：

- 每日签到时间
- 本地状态页端口
- 开机自动启动
- 启动时后台静默运行
- 关闭窗口时最小化到托盘

说明：

- “开机自动启动”开启后，程序会注册到当前用户的 Windows 启动项
- “启动时后台静默运行”开启后，程序启动时不会弹出主窗口，而是直接缩到托盘
- “关闭窗口时最小化到托盘”开启后，点击关闭不会退出程序，只会隐藏到托盘继续运行

## 5. 自动运行方式

项目支持三种常见运行方式：

### 方式一：手动启动 GUI

```bash
py -3 main.py tray
```

### 方式二：命令行直接签到

```bash
py -3 main.py
```

### 方式三：静默启动

```bash
py -3 gui_app.py --minimized
```

这个模式通常用于：

- 开机自启动
- 后台常驻
- 不希望开机弹出主窗口

## 6. 开机自启动逻辑

当你在“设置”页勾选“开机自动启动”后，程序会：

1. 在 Windows 当前用户启动项中写入应用启动命令
2. 下次登录系统时自动启动程序

如果同时勾选“启动时后台静默运行”，启动命令会附带静默参数，程序启动后会直接进入系统托盘。

## 7. 打包流程

开发完成后，可以将项目打包为单文件 exe：

```bash
build.bat
```

打包脚本会：

1. 安装或更新打包依赖
2. 调用 PyInstaller
3. 生成单文件 GUI 程序

输出位置：

```text
dist\GitHubCheckinTool.exe
```

## 8. 运行依赖

项目主要依赖：

- requests
- gitpython
- pystray
- Pillow
- schedule

安装命令：

```bash
py -3 -m pip install -r requirements.txt
```

## 9. 补充说明

- 配置文件保存在项目目录下的 `config.json`
- 开机自启动设置写入的是当前用户注册表，不影响其他用户
- 如果更改了端口，通常需要重启程序后才会完全生效
- 如果使用打包后的 exe，自启动也会直接指向 exe 本身

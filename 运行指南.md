# GitHub 每日签到工具 - 运行指南

## 一、环境要求

- **Python**: 3.8+
- **Git**: 已安装并配置好
- **GitHub**: 需要一个用于签到的仓库
- **系统**: Windows 10/11

---

## 二、安装

```bash
cd C:\Users\LIU\desktop\attendance
pip install -r requirements.txt
```

---

## 三、首次配置

```bash
python main.py config
```

配置过程会依次要求你填写：

### 1. AI 服务商配置
- 选择服务商（SiliconFlow / OpenAI / DeepSeek / 自定义）
- 确认或修改模型名称
- 输入 API Key

> 默认使用 SiliconFlow + GLM-4.7，密钥已在 `Qwen3-8B使用说明.txt` 中提供。

### 2. GitHub 配置
- 输入仓库地址（如 `git@github.com:username/daily-checkin.git`）
- 选择认证方式：
  - **SSH**: 需要提前配置好 SSH Key
  - **HTTPS + Token**: 输入 GitHub Personal Access Token（权限至少勾选 `repo`）
- 输入本地仓库存放路径（默认 `./repo`）

配置完成后保存到 `config.json`。

---

## 四、使用方式

### 方式一：GUI 桌面应用（推荐）

```bash
python main.py tray
```

启动后会打开一个桌面窗口，包含以下功能：

- **状态面板** — 查看运行状态、上次签到时间和结果、下次签到时间
- **签到选项** — 选择文章主题（随机或指定）、设置生成数量（1-10 篇）
- **操作按钮** — 立即签到 / 预览模式（只生成不推送）
- **设置区** — 每日签到时间、状态页端口、开机自启、最小化到托盘
- **运行日志** — 实时查看操作日志

关闭窗口后程序最小化到系统托盘继续运行，双击托盘图标恢复窗口。

#### 开机自启

在 GUI 设置区勾选「开机自动启动」并点击「保存设置」，程序会开机后自动以最小化托盘模式启动。

#### 状态页

启动后可在浏览器访问 `http://localhost:5000`（端口可在设置中修改）查看签到状态。

### 方式二：命令行

```bash
python main.py              # 执行一次签到并推送
python main.py --dry-run    # 只生成内容，不推送（保存到 output/ 目录）
python main.py test         # 测试 API 连接
python main.py config       # 重新配置
python main.py schedule     # 配置 Windows 计划任务（定时签到）
python main.py help         # 查看帮助
```

运行 `python main.py config`，选择不同的服务商。支持所有 OpenAI 兼容格式的 API：

- OpenAI
